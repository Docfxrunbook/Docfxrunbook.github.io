Site Permissions

By default, each SharePoint team site is part of an Microsoft 365 group. a Microsoft 365 group is a single permissions group that is associated with various Microsoft 365 services. When you add owners or members to the Microsoft 365 group, they're given access to the SharePoint site along with the other group-connected services. Permissions for sites can be managed through the Owners, Members, and Visitors groups, or through the associated Microsoft 365 group if there is one. Special permissions are needed to associate sites to a hub.

Managing the permissions of a hub site is dependent on the underlying type of site. If the site is a group-connected team site, then you should manage permissions through the Microsoft 365 group. If it's a communication site, then you should manage permissions through the SharePoint groups.

Giving people permissions to a site, group, or team gives them access to all site content.

Permissions granted to a user on a site gives them the ability to perform specific actions. For example, the View Items permission allows a user to view items in a list or folder, but not to add or remove items. Permissions can be granted to individual users at site or site content levels.
